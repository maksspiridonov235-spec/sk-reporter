# Блок «Отчёт инженера»

Инженер заполняет таблицу по ВОР за день → система подставляет выдержки из ТК →
генерирует **docx в том же формате**, что ежедневный отчёт СК → дальше в `/daily`.

## Папки

| Путь | Назначение |
|------|------------|
| `profiles/` | YAML-профиль инженера (`example.yaml`: проекты, путь к шаблону docx) |
| `launchers/` | `.bat` для выдачи инженеру (задаёт `SK_ENGINEER_PROFILE`) |
| `../sk_reporter/engineer/` | Python: парсер ВОР, каталог ТК, сборка отчёта |
| `../data/` | ВОР, ТК, персонал — см. `data/README.md` |

## Запуск (позже)

1. Техник кладёт данные в `data/projects/`, `data/tk/`, обновляет `work_tk_map.yaml`.
2. Создаёт `profiles/{id}.yaml` и `launchers/{id}.bat`.
3. Инженер запускает bat → браузер `/engineer` → заполняет → скачивает docx.
