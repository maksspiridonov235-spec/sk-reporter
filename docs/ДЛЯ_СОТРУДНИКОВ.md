# SK-Reporter — для сотрудников

## Кто что делает

| Роль | Действие |
|------|----------|
| **Ответственный за сервер** (один на офис) | Запускает ярлык SK-Reporter, следит за чёрным окном и Ollama |
| **Остальные** | Только браузер, ничего не устанавливают |

Подробная настройка ярлыка и батника: **[RUN_SERVER.md](RUN_SERVER.md)** → раздел «Офисный ПК: ярлык SK-Reporter.bat».

---

## Для ответственного: как запустить сервер

1. Один раз на ПК: `.\scripts\setup.ps1` (см. RUN_SERVER.md).
2. Создать ярлык на рабочем столе из `launcher\SK-Reporter.bat`.
3. Каждый день: **дважды щёлкнуть ярлык SK-Reporter**.
4. Дождаться браузера http://localhost:8000 — **чёрное окно не закрывать**.
5. Ollama должна быть запущена (иконка у часов) — для кнопок с AI.

---

## Для всех: работа в браузере

1. Открыть http://localhost:8000 (если не открылось само — спросить ответственного, запущен ли ярлык).
2. Загрузить отчёты (.docx) — перетащить или нажать на зону загрузки.
3. Выбрать **дату** в поле под списком файлов.
4. При необходимости: **Проверить и исправить**.
5. **Подготовить** → **Сформировать**.
6. Скачать готовые файлы справа или **Скачать (ZIP)**.

---

## Если не открывается

- Спросить ответственного: запущен ли ярлык SK-Reporter и не закрыто ли чёрное окно.
- Ollama в трее — нужна для кнопок с AI (проверка/исправление), не для простой склейки.
