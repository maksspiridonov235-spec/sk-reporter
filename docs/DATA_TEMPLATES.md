# Болванки шаблонов (data/templates)

## Где лежат

```
sk-reporter/data/templates/
├── README.md
├── Ежедневный отчет Шаблон.docx    ← обязателен для «Подготовить»
├── Евракор_Ежедневный отчёт СК за ….docx
├── ЮНС_Ежедневный отчёт СК за ….docx
└── … (остальные компании)
```

Код: `webapp/config.py` → `TEMPLATES_DIR = data/templates`.

## Политика Git

| Путь | В git? |
|------|--------|
| `data/templates/*.docx` | **Да** — болванки компаний |
| `output/*.docx` | Нет — исправленные отчёты |
| `%TEMP%/sk_reports_work/` | Нет — загрузки и результаты сессии |
| Прочие `*.docx` в проекте | Нет |

В `.gitignore`: `*.docx` с исключением `!data/templates/*.docx`.

## Миграция с `contractor_report/`

Раньше болванки лежали в:

`contractor_report/болванки (шаблоны не вырезать только копировать)/`

После обновления:

1. `git pull`
2. Убедитесь, что в `data/templates/` есть `.docx` (должно подтянуться из git)
3. Старую папку `contractor_report/` можно удалить

Если на ПC болванки только локально (не в git), **скопируйте** все `.docx` в `data/templates/` вручную.

Сервер временно подхватит старый путь, если `data/templates` пуст, но в логе будет `[WARNING]`.

## Добавление новой компании

1. Добавьте строку в `companies.py`
2. Положите болванку в `data/templates/` — имя файла должно содержать ключевое слово компании
3. Закоммитьте новый `.docx` в git (если обновляете репозиторий для всех ПC)

## Проверка

При старте сервера в логе:

```
[INFO] Templates dir: .../data/templates (N шаблонов)
```

Если `N` меньше ожидаемого — не хватает файлов болванок.
