## Текущий фокус (2026-06-08)

**Предписания** — проверка Excel в **Отчётность** (`/prescriptions`): B18/B19, Техэксперт, guard, отчёт в UI. Следующий шаг — проверка на офисном ПК с живым TE.

**Планирование** — справочники и ОТКК в PostgreSQL; **в работе** модуль «Расстановка / Прил.7» — см. [`docs/planning_deployment.md`](planning_deployment.md). Модуль ЛУВР **полностью снят** (2026-06-26).

Ежедневные отчёты СК (`/daily`) и отчёт инженера (`/engineer-hub`) — в эксплуатации, дорабатываются по запросу.

## Зачем проект

Автоматизация строительного контроля:

1. **Ежедневные отчёты** — загрузка docx, проверка описаний (Ollama), сборка в болванки подрядчиков.
2. **Предписания** — загрузка Excel, проверка замечания (B18) и нормативки (B19): Техэксперт / интернет + Ollama, guard, скачивание исправленного файла.
3. **Планирование** — персонал, проекты, ОТКК (PostgreSQL); **расстановка и Прил.7** — в разработке по [`docs/planning_deployment.md`](planning_deployment.md). Старый ЛУВР снят (2026-06-26).
4. **Отчёт инженера** — заполнение по ВОР и ТК.

## Целевой процесс планирования

| Шаг | Действие |

Тяжёлые xlsx/xlsm шаблоны — **локально на диске**, не в корне репозитория.

## Текущий процесс — ежедневные отчёты

Загрузка docx → дата + руководитель → **Проверить и сформировать** (check→inject, руководитель, макросы, сборка) → скачать ZIP.

## Текущий процесс — предписания

Главная → **Отчётность** → **Предписания** → загрузка xlsx/xls → **Проверить** → в отчёте: правки B18, источник нормативки (Техэксперт / интернет), вопросы инженеру, черновик письма → скачать исправленный Excel или ZIP.

Нормативная ссылка (B19) в файле: **краткий title документа из Техэксперта** (как в строке поиска) + пункты, которые указал инженер. Полное «Об утверждении ФНП…» система не подставляет.

Для сверки с TE на сервере нужны переменные `TE_EXPERT_*` — см. [`RUN_SERVER.md`](RUN_SERVER.md).

## Ограничения и среда

- **Продакшен:** RelaxDev — https://sk-reporter.relaxdev.ru, PostgreSQL (`DATABASE_URL`), Ollama cloud (`OLLAMA_API_KEY`)
- **Разработка** на Mac / локальный Windows — `uvicorn` из venv
- Тяжёлые xlsx/xlsm — **локально**, yaml-кэши — в git где уместно
