# Контекст проекта SK-Reporter

> AI: сначала [`memory.md`](memory.md) раздел **«СЕЙЧАС В РАБОТЕ»**, затем этот файл.

## Текущий фокус (2026-06-07)

**Планирование: справочники и назначения первыми**, затем расстановка, потом ЛУВР и Прил.7.

Ежедневные отчёты СК (`/daily`) и отчёт инженера (`/engineer`) — в эксплуатации, дорабатываются по запросу.

## Зачем проект

Автоматизация строительного контроля:

1. **Ежедневные отчёты** — загрузка docx, проверка описаний (Ollama), сборка в болванки подрядчиков.
2. **Планирование** — справочник персонала, проекты и назначения, расстановка; далее ЛУВР (учёт по дням), Прил.7.
3. **Отчёт инженера** — заполнение по ВОР и ТК.

## Целевой процесс планирования

| Шаг | Действие |
|-----|----------|
| 1 | **Персонал** — `personnel.yaml`, `person_id` |
| 2 | **Проекты** — `data/projects/*/project.yaml`, назначение инженеров |
| 3 | **Расстановка** — xlsm из справочника (кнопка в «Проекты») |
| 4 | **ЛУВР** — отметки по дням, помесячно (Excel и/или сайт) |
| 5 | **Приложение 7** — из шаблона xlsm + данные ЛУВР |
| 6 | *(опц.)* Расстановка из ЛУВР — если нужны только люди с отметками в месяце |

Шаблоны с макросами лежат в **`data/luvr/`** (см. `manifest.yaml`), не в корне репозитория.

## Текущий процесс — ежедневные отчёты

Загрузка docx → дата + руководитель → **Проверить и сформировать** (check→inject, руководитель, макросы, сборка) → скачать ZIP.

## Ограничения и среда

- Офисный **Windows**: один ПК, `launcher/SK-Reporter.bat`, Ollama для AI
- Разработка на **Mac**; `git pull` + перезапуск bat на офисе
- Тяжёлые xlsx/xlsm — **локально**, yaml-кэши — в git где уместно

## Что считается успехом (планирование)

- ЛУВР на сайте **как в Excel**, с правкой и выгрузкой
- Меньше ручного копирования в Прил.7 и расстановку
- Единый справочник персонала и привязка к объектам
