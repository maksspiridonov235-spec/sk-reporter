# 🤖 AI-Агент для анализа отчётов СК

Этот модуль использует **Ollama + модель Qwen** для интеллектуального определения компании в отчётах строительного контроля.

## 📋 Как это работает

1. **Быстрый поиск**: Сначала агент проверяет имя файла на наличие ключевых слов (как раньше).
2. **AI-анализ**: Если по имени файла компания не найдена, агент:
   - Открывает документ Word
   - Извлекает текст из первых 3 страниц (шапка, таблицы)
   - Отправляет текст модели **Qwen2.5:7b** через Ollama
   - Модель определяет название компании по содержимому
3. **Сопоставление**: Найденное название сверяется со списком известных компаний.

## 🔧 Требования

- Установленный **Ollama** (https://ollama.com)
- Модель Qwen: `ollama pull qwen2.5:7b`
- Python-пакеты: `ollama`, `requests` (уже установлены)

## 🚀 Настройка Ollama

1. Установите Ollama с официального сайта: https://ollama.com
2. Откройте PowerShell и выполните:
   ```powershell
   ollama pull qwen2.5:7b
   ```
3. Убедитесь, что сервис работает:
   ```powershell
   ollama list
   ```

## 📊 Преимущества

- ✅ **Не важно имя файла**: Даже если файл называется `asdf123.docx`, агент найдёт компанию внутри текста.
- ✅ **Понимает сокращения**: ООО, АО, ПАО, Групп, и т.д.
- ✅ **Устойчив к ошибкам**: Опечатки в названии компании внутри документа не помешают распознаванию.
- ✅ **Гибридный режим**: Если AI недоступен, система работает по-старому (по ключевым словам).

## ⚡ Быстрое тестирование

```powershell
cd agent
..\webapp\venv\Scripts\python.exe ocr_agent.py "путь/к/файлу.docx"
```

## 📝 Логи

При формировании отчёта в консоли сервера вы увидите:
- `🔍 По имени файла компания не найдена. Запускаю AI-анализ...`
- `✅ AI определил компанию: Евракор (ответ модели: Евракор)`
