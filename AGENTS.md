# SK-Reporter — инструкции для AI

Перед задачей в этом репозитории **прочитай** (если ещё не в контексте):

0. **`docs/memory.md` → раздел «СЕЙЧАС В РАБОТЕ»** — с чего начинать в новом чате (обновляется владельцем)
1. [`docs/memory.md`](docs/memory.md) — остальные решения и договорённости
2. [`docs/PROJECT_CONTEXT.md`](docs/PROJECT_CONTEXT.md) — продукт, люди, процессы, дорожная карта
3. [`docs/RUN_SERVER.md`](docs/RUN_SERVER.md) — запуск на Windows / Mac
4. [`docs/ДЛЯ_СОТРУДНИКОВ.md`](docs/ДЛЯ_СОТРУДНИКОВ.md) — инструкция для пользователей UI

## Роль проекта

Веб-приложение для **строительного контроля**: ежедневные отчёты (.docx), **планирование (ЛУВР)**, отчёт инженера по ВОР.

## Код

| Путь | Назначение |
|------|------------|
| `webapp/` | FastAPI + UI: `/`, `/planning`, `/reporting`, `/daily`, `/engineer-hub`, `/engineer` |
| `sk_reporter/` | docx, компании, агенты Ollama, `luvr_store.py`, `planning_data.py` |
| `sk_reporter/leader_switch.py` | смена руководителя в отчётах (rule-based) |
| `data/luvr/` | **ЛУВР**, luvr.yaml, шаблоны Прил.7 и расстановки (`manifest.yaml`) |
| `data/templates/` | Болванки подрядчиков |
| `data/projects/`, `data/tk/`, `data/personnel/` | Планирование |
| `engineer/profiles/`, `engineer/launchers/` | Профили и bat для инженеров |
| `sk_reporter/engineer/` | Парсер ВОР, каталог ТК, сборка отчёта инженера |
| temp `sk_reports_work/uploads/` | Загруженные и исправленные отчёты (рабочие; не в репо) |
| `launcher/SK-Reporter.bat` | Запуск сервера на офисном ПК |

## Правила работы агента

- Минимальный diff; не рефакторить без запроса
- Коммит и push — **только по явной просьбе** пользователя
- После изменения поведения для офиса — обновить `docs/memory.md` и при необходимости `docs/PROJECT_CONTEXT.md`
- Модель Ollama в агентах: `gemma4:31b-cloud` (см. код)
- После `git pull` на Windows сервер нужно **перезапустить** bat (код в памяти не обновляется)

## Обновление «памяти» между чатами

Факты из чата, которые должны помнить **все** будущие сессии Cursor, пиши в `docs/memory.md` (кратко, с датой). Не полагайся на историю другого чата в сайдбаре.
