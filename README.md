# SK-Reporter: Объединение отчётов строительного контроля

Веб-приложение для автоматического объединения ежедневных отчётов в сводные документы по компаниям.

## Функции

✅ **Интеллектуальный поиск компаний**
- Поиск по имени файла (быстро)
- AI-анализ содержимого документа (если имя не совпадает)
- Использует локальную модель Qwen2.5:7b через Ollama

✅ **Объединение документов**
- Автоматическое объединение отчётов в шаблон
- Сохранение форматирования и макросов

✅ **Макросы для обработки**
- Выделение 2-й строки таблиц
- Форматирование документов
- Установка дат (сегодня/вчера)

✅ **Удобный веб-интерфейс**
- Загрузка файлов с Drag & Drop
- Просмотр загруженных файлов
- Скачивание готовых отчётов
- Логирование операций в реальном времени

## Установка

### 1. Требования

- **Python 3.9+**
- **Ollama** (для AI-функций) - [https://ollama.com](https://ollama.com)
- **Git** (опционально)

### 2. Установка модели Ollama

```powershell
ollama pull qwen2.5:7b
```

### 3. Установка Python зависимостей

```powershell
cd C:\Users\Anton\Desktop\sk-reporter\webapp
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Также установите зависимости агента:

```powershell
pip install -r ../agent/requirements.txt
```

## Запуск

### Способ 1: Командная строка (Windows PowerShell)


Затем откройте браузер: **http://localhost:8000**

- Нажмите F5 или выберите Run → Start Debugging
- Конфигурация хранится в `.vscode/launch.json`

## Использование

1. **Загрузите отчёты** от инженеров (.docx)
2. **Загрузите шаблоны** для каждой компании (.docx)
   - Имена должны содержать ключевые слова: "Евракор", "ЮНС", "НГСК" и т.д.
3. **Сформируйте отчёты**:
   - Кликните на компанию для одного отчёта
   - Или нажмите "СФОРМИРОВАТЬ ВСЕ ОТЧЁТЫ"
4. **Скачайте результаты** из раздела "Готовые файлы"

## Список компаний

Система работает со следующими компаниями:

- **Евракор** (евракор, еврокор)
- **Лесные технологии** (лесн. технологии, лесные технологии)
- **ЮНС** (нткс, юнс, югранефтестрой)
- **НГСК** (нгск, ткс)
- **Сибитек** (сибитек, ооосибитек)
- **ЭСМ** (эсм, энергостроймонтаж)
- **НГП**, **РОСЭКСПО**, **ТПС**, **ТЭКПРО**, **ЮПИ**, **УГГ**, **ЮГС**, **ТВС**, **НСС**, **ОТ и ТБ**, **Стройфинансгрупп**

Ключевые слова для поиска находятся в `webapp/docx_processing.py` в переменной `COMPANIES`.

## API Endpoints

### GET endpoints

- `GET /` - Главная страница
- `GET /files/reports` - Список загруженных отчётов
- `GET /files/templates` - Список загруженных шаблонов
- `GET /results` - Список готовых файлов
- `GET /download/{filename}` - Скачать файл

### POST endpoints

- `POST /upload/reports` - Загрузить отчёты
- `POST /upload/templates` - Загрузить шаблоны
- `POST /merge/{company}` - Объединить отчёты компании
- `POST /merge/all` - Объединить все отчёты
- `POST /macro/{name}` - Применить макрос
- `POST /rename/{mode}` - Переименовать файлы

### DELETE endpoints

- `DELETE /clear/reports` - Удалить загруженные отчёты

## Технические детали

### Стек

- **Backend**: FastAPI + python-docx + Ollama
- **Frontend**: HTML5 + CSS3 + Vanilla JavaScript
- **AI**: Ollama + Qwen2.5:7b
- **Storage**: Временная папка Windows (%TEMP%)

### Как работает AI-анализ

1. Если имя файла содержит ключевое слово (быстро) → используется это значение
2. Если не найдено → извлекается текст из первых 3 страниц документа
3. Текст отправляется на локальную модель Qwen2.5:7b
4. Модель определяет компанию из известного списка
5. Результат кэшируется на сессию

### Безопасность

- Все файлы хранятся во временной папке Windows (`%TEMP%\sk_reports_work\`)
- Файлы НЕ сохраняются между перезапусками сервера
- Нет аутентификации (приложение для локального использования)
